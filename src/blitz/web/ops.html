Mode 24/7 — supervision

Faites tourner StormCell en continu pour tout capturer. Cette page surveille le flux, la base et le disque en direct, et pilote la robustesse (re-sélection de serveur, rétention, maintenance, sauvegarde). La capture est indépendante du navigateur : une fois le serveur lancé, tout est enregistré même cette page fermée.

Lancer au démarrage & redémarrage automatique

Le redémarrage automatique du processus se règle au niveau du système d'exploitation — cette page ne peut pas le faire à votre place. Choisissez une méthode :

# docker-compose.yml — redémarre tant que vous ne l'arrêtez pas
services:
  stormcell:
    build: .
    restart: unless-stopped
    ports: ["8000:8000"]
    volumes:
      - ./data:/app/data        # la base et les logs survivent aux redémarrages
    environment:
      - BLITZ_DB=/app/data/lightning_log.db

# puis :  docker compose up -d
# NSSM — installe StormCell comme service Windows (auto-restart + démarrage au boot)
nssm install StormCell "C:\chemin\.venv\Scripts\python.exe" "-m blitz web"
nssm set StormCell AppDirectory "Z:\Projets\StormCell"
nssm set StormCell AppExit Default Restart
nssm set StormCell Start SERVICE_AUTO_START
nssm start StormCell
# Watchdog — relance si le flux est mort depuis > 5 min (PowerShell, tâche planifiée /5 min)
$s = Invoke-RestMethod http://127.0.0.1:8000/api/ops/status
$age = (Get-Date -UFormat %s) - $s.last_message_at
if ($age -gt 300) {
  Restart-Service StormCell        # ou: docker compose restart stormcell
}

À surveiller pour du vrai 24/7 : l'espace disque (carte « Disque » ci-dessus), et le fait de ne lancer qu'une seule instance sur la même base de données.
